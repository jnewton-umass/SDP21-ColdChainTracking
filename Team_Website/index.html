Cold Chain Tracking

Team 25:

Cam Harvey, Calvin Lee, and John Newton

Problem Statement:

Our original Problem Statement portrays the need to monitor temperature and location during transit for accountability purposes. However, Current solutions are not waterproof, expensive, or single use only.

Solution:

  • Single gateway device for truck with cellular and gps communication
  • Internal temperature sensors all report to gateway for transmission of temperature and location data to the web server
  • Internal light sensor to help explain discontinuity
  • QR codes on each product detailing origin, destination, product number
  • QR code on gateway/truck to link measurements
  • Mobile scanner application to store and upload to server
  • Shipments will be scanned during loading and unloading
  • QR info will correlate records with authorized parties

System Specs:

  • Power: Must run off of 12V lead acid source (truck battery)
  • Must be able to provide real time tracking to user with the following parameters:
    • Temperature: within 1 degree Fahrenheit
    • Location: within 25 ft
    • Light: Above or below a set threshold (daylight/darkness)
    • Time: Record the correct time in Eastern Standard Time within 1 sec
    • Updated every 15 minutes with connection
  • Cost: Less than $284 for materials for final prototype
  • Size: Gateway no larger than 6”x6”x3”
